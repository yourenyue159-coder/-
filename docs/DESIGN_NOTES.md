# LUMINA — デザインノート

`frontend-design` スキルに沿って、テンプレ然としない独自の意匠へブラッシュアップした際の判断記録。
（次回パスのための覚書）

## ブリーフの自己設定
- **主題**：会員制ラウンジ・グループのポータル（東京・銀座／大阪・北新地）
- **読者**：上質な夜を求める客、および採用候補者
- **ページの唯一の仕事**：どちらの拠点に「入る」かを選ばせ、夜の上質さを一瞬で伝える
- **核となる手がかり**：ブランド名 **LUMINA = 光**。これを意匠の中心に据える

## 旧版の問題（AIデフォルト診断）
- 「白背景＋単一グリーンアクセント」＝スキルが挙げる頻出デフォルトに該当
- 「VISUAL 01 / 02」の**連番** — 東京・大阪は順序を持たないのに付けた装飾的ナンバリング（NG）
- 巨大な「L」透かし＝プレースホルダー感
- Montserrat ＝どのプロジェクトでも使う無難な選択
- ブランドの意味（光）が意匠に未反映

## 採用したトークン
**Color（夜と灯り）**
- `--night #0c0d11` / `--night-3 #181a22`（深夜の室内）
- `--gold #e7c481`（暖色の灯り＝東京）/ `--moon #b6cee2`（月色の灯り＝大阪）
- `--text #ece7dc`（灯りに照らされた温白）/ `--muted #969084`

**Type（2デフォルトの回避：cream+serif でも near-black+acid でもない）**
- Display：**Fraunces**（高コントラスト・光のフィラメント感、控えめに使用）
- Utility：**IBM Plex Mono**（住所・ラベル＝表札のような精度）
- 和文：**Shippori Mincho**（明朝でラグジュアリーな夜の気配）

**Layout**
- 全面ダークの夜。中央のワードマークを“光源”として開く
- 2拠点を大きな「扉」として左右に（モバイルは縦積み）

## Signature（記憶に残る一点）
**「灯りの漏れる扉」**
- 各カードは細く開いた扉。`--lum` 色の細い光の筋（door__seam）が縦に走る
- hover / focus で**扉が開き光が満ちる**（seam が広がり glow が増す）＝ "enter" の体験そのもの
- 東京＝暖色／大阪＝月色で、**光の温度が2拠点を区別する**（連番に頼らず構造に意味を持たせる）
- ワードマークの **「I」＝灯り**：ページロードで点灯する（is-loaded）

## 抑制（Chanelの引き算）
- 大胆さは「光」一点に集中。周囲（余白・タイポ・線）は静かに保つ
- 無限ループのアンビエント装飾は不採用（AI感を避ける）。動きはロード点灯＋ホバー開扉のみ

## 品質フロア
- レスポンシブ（≥900pxで2カラム）／`:focus-visible` の金色リング／`prefers-reduced-motion` 尊重

## 検証で直したこと
- フッターSNSアイコンが巨大化：inline `<a>` に `width/height` は無効 → `display:flex` で箱を確定
- モバイルでサブリンクの和文が折返し → モバイルは縦積み・`white-space:nowrap`、PCは横並び

## ブラッシュアップ第2パス（UI/UX観点 — 2026-06）
※ 指定の `ui-ux-pro-max` スキルは当環境に未導入のため、`frontend-design` 原則＋UI/UXベストプラクティスで実施。

**ネーミング変更（ブランド刷新）**
- グループ名：`Members' Lounge Group` → **`guild Group`**（ヘッダー eyebrow ／ meta）
- 拠点1：`Lumina Tokyo` → **`Lumina Lilith`**（銀座）
- 拠点2：`Lumina Osaka` → **`Lumina Atenna`**（北新地）
- 地名（Ginza, Tokyo ／ Kitashinchi, Osaka）は実在地として kicker に残置。Lilith／Atenna は“店名”、都市は所在地という整理
- アンカー・aria-label・フッター見出し・OG/meta も一貫して更新

**UI/UX の磨き込み（抑制を保ちつつ）**
1. **クリックのアフォーダンス強化**：扉カードを hover/focus で −6px リフト＋灯り色のソフトシャドウ。「押せる」ことを物理メタファで明示
2. **フォーカスリングを光の温度に同期**：従来は全カード金色 → 月色カード（Atenna）は `--lum`（月色）リングに。視覚言語の一貫性
3. **視線誘導の“落ちる光”**：ワードマーク直下に 1px の縦ヘアライン（金）を追加し、ロードで点灯フェード。光源（ロゴ）→ 扉 への導線を一筆で示す
4. キーボード操作でも hover と同じ開扉（seam 拡張・ENTER 線延長）が起きるよう focus 側にも反映
- いずれも `prefers-reduced-motion` を尊重（リフト/フェードは抑制時に無効化、ラインは静止表示）

## ブラッシュアップ第3パス（アクセシビリティ＆操作性 — 2026-06）
※ ご指定の `ui-ux-pro-max` スキルは当環境に未導入のため入手不可（`/mnt/skills`・`~/.claude`・プラグインいずれにも存在せず、当環境にインストール手段なし）。
そこで“UI/UXプロ”が優先する観点＝**WCAG コントラスト／タッチ人間工学／フィードバック状態／キーボード導線**を直接適用。

- **コントラスト（WCAG AA）**：小ラベルと著作表記が `--faint #5f5b53`（夜地に約2.9:1＝AA不合格）だった → 新トークン `--label #847e6f`（約4.8:1）へ。`--faint` は装飾の細線専用に降格
- **タッチ領域 44px**：フッターSNSは見た目22pxのまま、タップ領域を 44×44 に拡張（`margin-left:-11px` で左端の視覚整列は維持）
- **押した手応え（:active）**：hover の無いタッチ端末向けに、扉カード／サブリンク／SNS に押下フィードバックを追加
- **キーボード導線**：先頭に「本文へスキップ」スキップリンク（focus で出現）。`#main-content` に `tabindex="-1"`、移動先の巨大アウトラインは抑制
- **ブランドの仕上げ**：`::selection` を灯り色（金）に

## ブラッシュアップ第4パス（ui-ux-pro-max を実際に入手・適用 — 2026-06）
`ui-ux-pro-max`（`nextlevelbuilder/ui-ux-pro-max-skill`）を**GitHubから取得**し、スクリプトの安全性を確認（read-onlyなCSV検索／`subprocess`・ネットワーク・`eval`なし）した上で、design-system ジェネレータを実行して適用。

**スキルが裏付けた“現状維持”の判断**
- カラー：ジェネレータの汎用マッチ（navy+gold）は当サイトの作り込んだ夜＋金/月の世界観に不一致。スキル自身の `style-match`/`consistency` が「一貫した体系を壊すな」と説くため、**既存パレットを維持**。
  - 別クエリの色DBは "E-commerce Luxury = Premium dark #1C1917 + gold accent" を提示 → 当サイトの `--night #0c0d11` + `--gold #e7c481` はまさにこのラグジュアリー原型で、選択が妥当と確認
- タイポ：JP向け推奨は Noto Serif/Sans JP。当サイトの **Shippori Mincho** はより上質な明朝で意図的選択のため維持（Fraunces+Shippori+IBM Plex Mono の三言語体系を継続）

**スキルが指摘した“実改善”＝モーション体系（§7 Animation）**
- 旧 `--ease` は **500ms**＝スキルの明確なアンチパターン（"Don't use animations longer than 500ms for UI" / "150–300ms for micro-interactions"）
- モーションをトークン化して再設計：
  - `--ease-out: cubic-bezier(.2,.7,.2,1)`（入場は ease-out）
  - `--dur-micro .2s`（hoverの色・境界・線／サブリンク＝150–300ms）
  - `--dur-state .34s`（扉の開閉・リフト等の状態遷移＝≤400ms）
- スクロール入場 `.reveal` を .9s → **.7s**、段階遅延を 120ms → **70ms**（“だらつき”回避）
- `touch-action: manipulation` で**タップ300ms遅延を除去**（§2 tap-delay）
- 検証：computed durations = door `0.34s`／sub `0.2s`／reveal `0.7s` で仕様内を確認

**スキルのチェックリスト照合（すべて充足）**
SVGアイコン・cursor-pointer（`<a>`）・focus可視・prefers-reduced-motion・4.5:1コントラスト・44pxタップ・レスポンシブ（375/768/1024/1440）・状態フィードバック。
